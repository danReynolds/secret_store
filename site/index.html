Local secret storage

Secrets stay local.

Keybay stores secrets behind the operating system's keystore and gives them only to the Dart code or process that asks. No account, server, daemon, or network path.

Use from source Threat model

.secrets.env

API_URL=https://staging.example.com
OPENAI_API_KEY=kb://acme/openai-api-key

$ keybay set acme/openai-api-key
$ keybay run -- npm start

Two surfaces

One storage model.

Use the SDK when Dart can read directly. Use the CLI when an existing application already reads environment variables.

Dart SDK

Read secrets in code.

One constructor selects the platform path. Values are bytes at the core, with string conveniences.

pubspec.yaml · pre-release

dependencies:
  keybay:
    path: ../keybay

Dart

import 'package:keybay/keybay.dart';

final store = SecretStorage(
  appId: 'com.example.myapp',
);

final token = await store.readString('api-token');

Five-command CLI

Inject secrets at launch.

A committed manifest names what the app needs. Keybay resolves it, injects one process, and exits from the process tree.

Shell · pre-release

git clone https://github.com/danReynolds/keybay.git
cd keybay
dart pub get
dart pub global activate --source path packages/keybay_cli

run set rm list doctor

Security

Fail closed or do not run.

Keybay chooses one supported path per platform. If that path is unavailable, locked, invalidated, or corrupt, it returns a typed failure instead of selecting a weaker home.

Local
No account, network service, sync protocol, or resident process.
Authenticated
XChaCha20-Poly1305, key commitment, atomic writes, locking, and fsync for the file path.
Measured
Hardware backing is reported only after platform-specific inspection.
Auditable
One third-party runtime dependency and real-keystore platform tests.

Windows and headless systems are not supported. Keybay does not silently fall back.

Scope

Less is the feature.

Keybay is for local storage and run-scoped injection. It is deliberately not a universal secrets platform.
